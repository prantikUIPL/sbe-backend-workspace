## Summary

Read-only exhibitor Order Details page. The core data is well-backed: the exhibitor cart checkout write-path (exhibitor-backend-api cart.service.ts:1229-1305 + cart.helpers.ts materializeCartToOrder) writes Order (subtotal, coupon_amount, setup_fees, cleaning_fees, total_savings, total, paid_amount=0, billing), OrderItems carrying cart_item_type (booth/sponsorship/addon/fee), description, quantity, unit_price, amount and parent_order_item_id, plus OrderAgreement (signer, signature URL, terms_accepted, signed_at) and a processing PaymentTransaction; the external-api webhook (webhook.service.ts:2042-2098 + buildPtSuccessUpdate:2187-2201) flips the order to paid, sets paid_amount/charge id/paid_at, and creates Invoice + InvoiceLineItem tree for cart-flow orders. So the net-new `GET /orders/:orderId` aggregate and its header/booth/line-item/financial/payment/agreement-acceptance/ownership/reconciliation fields are **Deliverable** on built primitives (ownership enforced via existing auth guards + company_id scoping) — that is the 19-requirement confirmed build scope. Booth Size is Deliverable (joinable structured data: booth OrderItem.product_id -> Product.length/width, the canonical 10x10). **Revised verdicts (2026-07-01, per-item review):** the invoice-PDF download (13.3-s) is **Deferred** pending the invoice A-vs-B decision. The multi-show header shape (13.3-b) was **RESOLVED** — per-show list (array of event/city/dates) — so 13.3-b is Deliverable. The signed-agreement download (13.3-v) **already exists in code** (`GET /agreement/signed/:orderId/download`, authored by Kshitiz under SBE-862/869) and covers booth/product orders — now **Deliverable** (FE wiring only; DOCX format, `is_manual_bypass` and SBE-1158 reconcile are non-blocking confirmations), which also resolves the compound 13.3-ac. Gift Certificate (13.3-l) — **re-verified 2026-07-05**: a redemption producer now exists (`captureGiftCertificateForOrder` in the webhook writes `GiftCertificateRedeem`, and the Order snapshots `gift_certificate_purchase_id`/`code`/`discount` at checkout), so 13.3-l was **shipped** as a gift-certificate line in the financial summary (PR #303). Booth Upgrade (13.3-g, Blocked on the Booth Upgrade story), salesperson Payment Schedule (13.3-t, Blocked on SBE-1154 / OM 24.8), and Onsite Contact (13.3-w, Blocked on SBE-1169/1165, Sprint 5) all have owning stories rather than being flatly out of scope. Only Booth Number (13.3-c — assigned externally after purchase) and Booth Type (13.3-f — no classification data exists) remain **Not Deliverable**, both parked as open questions.
