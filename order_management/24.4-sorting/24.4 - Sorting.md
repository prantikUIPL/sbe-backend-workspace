## Summary

Story 24.4 asks for admin-side sorting of the Order Management list on the Order and Date columns (ascending/descending), enforced on the backend across the full dataset rather than the current page. There is no order-management module yet, so the list endpoint is net-new, but the underlying data and the sorting mechanism are fully built: the Order model carries order_number (@unique, written at order creation) and created_at (@default(now())) as concrete, indexed scalar columns, and admin-backend-api already implements backend sorting via sortBy/sortOrder query params translated to Prisma orderBy (e.g. coupon-codes, ppl-order). Building sort onto the net-new GET /admin/orders endpoint on these working primitives is straightforward, so every requirement is Deliverable. No cross-epic dependency and no true ambiguity gates this story.
