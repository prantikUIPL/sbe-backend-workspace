## Summary

Admin Payment Plan screen over an order's PaymentTransactions: view all installments (8 columns), edit future dates, mark Paid/Unpaid/Refund, delete/add installments, post-payment lock, derived Unallocated Balance, 60/30-day rules with warn-not-block bypass, milestone statuses. Re-verified 2026-07-03 on origin/dev (admin `0ac2769`): all four routes are net-new **additive routes on the shipped orders module** (stale "no order module" premise dropped; the shipped controller has exactly list + notes routes). Verified fresh: **no manual payment flow exists anywhere** — manual methods appear only as *intent* on the display-only cart schedule (`CartPaymentScheduleEntry.method`); all 5 PT creation sites are card flows (read in full at the gate); no mark-as-paid handler exists; the webhook is the only `succeeded` writer. 24.8/24.7 introduce manual payments to the system for the first time.

**Decisions locked 2026-07-03:** (D2) add `payment_type` to payment_transactions **reusing the `CartPaymentMethod` enum** (`credit_card|bank_wire_ach|check|paypal` — exactly the sheet's taxonomy), **backfill `credit_card` + NOT NULL** (provably safe: all historic rows are card rows), **no lingering DB default** — all **7 pre-existing card writers** updated in-release to set it explicitly (exhaustive sweep 2026-07-06: exhibitor ×4, external ×2, admin ×1; the gate's "5 sites" undercounted by two — exhibitor `ppl.service.ts` and external `webhook.service.ts` upsert); `payment_memo` nullable Text, service-enforced mandatory for non-CC; **schema doc-comments in BOTH places** (the enum declaration and the new column): `credit_card` = Stripe card charge, the sole non-manual value; cart-copy enrichment (copy per-installment type from the cart schedule for cart-born orders) noted for plan time. (D3) make `stripe_payment_method_id`/`stripe_customer_id` **nullable** (doc-comment: null = manual non-card installment — no Stripe identity exists), keep `idempotency_key` NOT NULL via app-generated UUID for manual rows; **mandatory cron guard**: worker charge + retry queries add `payment_type: credit_card` so manual rows are never charged. (D4) new `PaymentTransactionSource.admin_manual` for admin-added installments (SBE-421's `admin_change_plan` precedent). (D1) chargeback milestone statuses are **display-only** here — see 24.8-l split.

Refund execution is **delegated to 24.9** (same release, builds first): the plan screen's Refund action calls 24.9's per-installment refunds endpoint; 24.8 owns no refund code.
