## Summary

Story 24.3 asks for backend-enforced search of the Order Management list by customer name, company name, or order ID, with partial and case-insensitive matching and an empty state on no match. Re-verification against actual code confirms every primitive is built and populated: Order.order_number (unique VarChar, written on every create path) and the numeric id PK; Company.name (non-null) plus a guaranteed company relation (company_id non-null FK) and the billing_company_name snapshot; and billing_first_name/billing_last_name snapshots written by exhibitor cart (cart.service.ts:1248-1249), PPL checkout (ppl/checkout.service.ts:186-187), and admin cart checkout (admin/cart/checkout.service.ts:264-265). A near-identical search already runs in admin ppl-order (insensitive contains over order_number + company.name, ppl-order.service.ts:296-302) returning a paginated data+meta shape. The Order Management list endpoint is net-new, but all primitives exist, so all 7 requirements are Deliverable. The only true open item is the spec's own \"subject to client confirmation\" note on order-ID match semantics (substring order_number vs exact numeric id). Note: the existing ppl-order list is scoped to subscription/ppl_addon order types and does NOT yet search customer name, so the customer-name clause is net-new (trivially backed by written data).

## Open Questions

- **[OPEN — for team discussion]** Spec explicitly defers match behaviour to client confirmation: order-ID search may mean substring+insensitive over the human-readable order_number (consistent with the partial-match requirement) OR exact match on the numeric id PK. Customer/company-name targets are unambiguously partial+insensitive.
  - **Current direction (2026-07-01):** lean toward substring+insensitive `contains` over `order_number` (consistent with 24.3-d's partial-match requirement and the live ppl-order precedent), but hold for client confirmation before building — the numeric-id exact-match interpretation would change the where-clause. — (pending)
