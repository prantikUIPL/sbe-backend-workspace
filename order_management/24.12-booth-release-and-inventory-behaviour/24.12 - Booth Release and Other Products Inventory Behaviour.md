# Story 24.12 — Booth Release and Other Products Inventory Behaviour

**Epic:** Order Management

**Confluence:** [Order Management → Story 24.12](https://unifiedinfotech.atlassian.net/wiki/spaces/SBE/pages/3859742741/SBE+-+Admin+Panel#Order-Management) _(page section; Confluence has no per-requirement anchor)_

## Summary

This story requires that when an Admin cancels an order in Order Management, the booth(s) and all other products on that order are released back to available inventory on the backend, with a manual-release reminder note shown in the cancel popup where automatic floorplan (Social Tables) release is unsupported. The codebase has a fully-built, proven inventory-release primitive: reservations flip from committed to released (with released_at) and availability is derived as ShowProduct.quantity − Σ(committed); this exact release is already wired into the Stripe payment-failure/cancel webhooks, and both booth and non-booth stock-bearing lines create committed reservations at checkout. So the SBE-internal release of booths and other products on cancel is Deliverable as a net-new admin cancel endpoint reusing existing code. The one genuinely blocked piece is AUTOMATIC floorplan release via Social Tables: only a stored display URL (Show.floor_plan_link, example diagram.socialtables.com) exists — there is NO programmatic Social Tables release API/integration — which the story itself flags as subject to confirmation and covers with the manual-reminder fallback (the fallback note is Deliverable).
